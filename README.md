# FlatCraft 2D - Процедурный генератор миров

## Идея проекта
Создание 2D-песочницы с видом сверху. Основа проекта — движок процедурной генерации карты на базе фрактального шума. Алгоритм создает логичные биомы - океаны, пустыни, леса, зимние биомы.

## Выбранные внешние библиотеки
1.  **Участник 1:** `perlin-noise` + `NumPy`
    *   Библиотека используется для генерации многооктавного фрактального шума. На её основе будут создаваться тепловые карты и карты влажности, пересечение которых формирует биомы. `NumPy` необходим для векторизации сложных математических вычислений шума на матрицах большого размера без использования циклов `for`.
2.  **Участник 2:** `Arcade`
    *   Аппаратно-ускоренный OpenGL фреймворк для 2D-графики. Будет использован для управления слоями графики, создания интерактивной камеры и оптимизированного рендеринга тысяч тайлов одновременно.

## Распределение фичей по итерациям

### Итерация 1 (Базовый движок и макро-мир)
**Участник 1 (Логика и генерация):**
1.  **Ядро генератора мира:** Написание модели получения 2D-массива биомов (матрицы) с использованием фрактального шума и `NumPy`.
2.  **Архитектура данных:** Создание предметной области: классы `World`, `Chunk`, `Tile`. Использование `dataclass` для хранения свойств тайлов. Настройка строгой типизации.

**Участник 2 (рендер и движок):**
1.  **Игровой цикл и камера:** Инициализация окна `Arcade`. Реализация камеры, позволяющей перемещаться по сгенерированной карте и изменять масштаб.
2.  **Базовый рендер слоев:** Чтение матрицы мира и отрисовка её на экране с разделением на два слоя: "Земля" и "Объекты" (пока в виде базовых геометрических фигур).

### Итерация 2 (Автотайлинг, текстуры и оптимизация)
**Участник 1 (Алгоритмы):**
1.  **Алгоритм автотайлинга:** Написание алгоритма, который анализирует соседей каждого тайла, высчитывает битовую маску и назначает правильную текстуру для создания плавных переходов (берега рек, закругления песка) без жестких границ.
2.  **Парсер конфигурации мира:** Вынесение настроек генерации (настройки шума, уровень моря и т.п.) в JSON-файл. Использование `pydantic` для строгой валидации конфига перед стартом генерации.

**Участник 2 (Графика и UI):**
1.  **Рендеринг и спрайт-листы:** Загрузка реальных текстур. Сбор метрик FPS для отчета по профилированию.
2.  **Интерактивный UI:** Интеграция модуля `arcade.gui` для создания меню (миникарта, настройка seed'а мира, мониторинг производительности).
